# AgentLinear 前端逻辑冻结说明

本文记录进入后端开发前已经确认的产品规则。后端的数据模型、进程管理和接口必须服从这些交互，不反向修改前端语义。

## 1. Session 分组

- 一个分组关联一个本地文件夹。
- 分组标题使用文件夹 basename。
- 支持创建、重新关联、重命名和移除空分组。
- 非空分组不能直接移除。
- 分组只负责组织任务，不参与调度优先级。

## 2. 任务与会话

- 一张卡片对应一个长期 Session。
- 首轮指令和后续消息都可以携带多个本地文件。
- 续聊保留完整历史、附件元信息和上下文轮数。
- 已完成、失败和已取消任务都能继续沟通。

## 3. 状态机

| 当前状态 | 允许进入 |
| --- | --- |
| 草稿 | 运行、排队、取消 |
| 排队 | 运行、取消 |
| 运行 | 完成、失败、取消、重新排队 |
| 完成 | 运行、排队 |
| 失败 | 运行、排队、取消 |
| 取消 | 运行、排队 |

非法转换不修改任务数据。卡片、抽屉和容量统计使用同一状态源。

## 4. 调度

- 全局并发上限固定为 6。
- 等待任务使用独立的 `queueOrder`，不依赖数组位置。
- 所有分组共享同一个先进先出（FIFO）队列。
- 任务完成或停止后，最小 `queueOrder` 的任务获得空闲槽位。
- 已取消任务永远不会被递补。
- 完成任务续聊时排到现有队列末尾，不允许插队。

## 5. 完成与提醒

- 任务完成后产生最终交付和一次性未读高亮。
- 声音提醒可以关闭，设置保存在本地。
- 同一完成事件只提醒一次。
- 打开任务后清除未读高亮。

## 6. 持久化与恢复

- 当前前端状态包版本为 v3。
- 保存任务、消息、附件元信息、队列顺序、分组、声音设置和最后打开页面。
- 页面重开后，原运行任务显示“等待恢复”，不占执行槽位。
- 恢复时有空位就运行，满载时进入队尾。
- 损坏数据安全回退到演示初始值，不产生白屏。

## 7. 后端接入边界

后端阶段需要替换四个模拟层：

1. Codex Session 创建、恢复与最终结果。
2. 子进程启动、进程树停止和崩溃检测。
3. SQLite 持久化与事务恢复。
4. 本地文件选择、目录重命名和附件读取。

前端状态名、队列规则、上下文连续性和危险操作确认保持不变。
