# AgentLinear 前端验收脚本

## 准备

1. 打开 `index.html`。
2. 确认侧边栏包含“任务看板”和“需求排期”。
3. 在需求排期中确认 11 项需求均显示“已完成”。

## 完整流程

1. 回到任务看板，确认现有分组显示本地文件夹名称。
2. 点击“添加分组”，选择一个本地文件夹；新分组出现在看板末尾。
3. 打开分组三点菜单，修改名称并检查路径预览同步变化。
4. 在新分组中创建任务，填写标题和指令，添加两个附件后删除其中一个。
5. 连续创建任务，直到页面显示 6 个运行任务，其余任务按序排队。
6. 打开一张运行卡片，追加文字和附件；确认历史消息仍在。
7. 点击“停止任务”，先取消确认，再重新操作并确认；任务变为已取消。
8. 确认队首任务自动开始，其他队列序号连续重排。
9. 打开运行任务并点击“演示完成”；确认卡片出现完成高亮和一次提醒。
10. 打开完成卡片；确认最终交付可见且高亮消失。
11. 刷新页面；确认分组、任务、消息、附件元信息和队列顺序仍在。
12. 确认刷新前运行中的任务显示“等待恢复”，点击“恢复任务”后重新占用槽位或进入队尾。

## 验收结果

- 所有步骤通过：前端逻辑可以冻结，进入后端设计。
- 任一步失败：记录对应 GitHub Issue，修复后从失败步骤重新执行。
